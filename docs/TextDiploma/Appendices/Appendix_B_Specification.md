Приложение Б (рекомендуемое) Спецификация атрибутов таблиц

Спецификация составлена по физической модели данных PostgreSQL (схемы `auth`, `deal`, `payment`), реализованной скриптами каталога `sql/postgres/`. Приведены ключевые таблицы, соответствующие основным сценариям системы: учётные записи, каталог санаториев, бронирования, выставление счёта и проведение платежа. Вспомогательные таблицы (`auth.refresh_tokens`, `deal.medical_profiles`, `deal.sanatorium_medical_profiles`, `deal.contracts`) в приложение не включены. В таблицах указаны роли атрибутов: PK — первичный ключ, FK — внешний ключ.
